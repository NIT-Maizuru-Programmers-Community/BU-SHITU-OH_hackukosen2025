# タスク 05: レース機能

## 概要

毎日開催されるレース（ベット機能）を実装する。固定のキャラクターが競争し、ランダムで勝者が決まるギャンブル要素で、ベットにより盛り上げる。

**重要**: レースに参加するのはユーザーではなく、固定の**キャラクター**です。ユーザーはキャラクターの中から勝つと思うものを選んでベットします。

## Firestore スキーマ

```typescript
// races コレクション
interface Race {
	id: string;
	date: string; // "2025-12-10"
	status: "open" | "closed" | "completed";
	characters: RaceCharacter[]; // レース参加キャラクター
	winnerCharacterId?: string; // 勝者のキャラクターID（完了後）
	results?: RaceResult[]; // 順位結果（オプション）
	totalBetPoints: number; // 総ベットポイント
	finalOdds?: number; // 確定オッズ（完了後）
	scheduledStartTime: Timestamp;
	endTime?: Timestamp;
	completedAt?: Timestamp;
	createdAt: Timestamp;
	updatedAt: Timestamp;
}

interface RaceCharacter {
	characterId: string; // キャラクターID（例："rabbit", "turtle"）
	name: string; // キャラクター名（例："ウサギ"）
	emoji?: string; // 表示用絵文字（例："🐰"）
	totalBets: number; // このキャラクターへの総ベット数
	totalBetPoints: number; // このキャラクターへの総ベットポイント
	rank?: number; // 順位（完了後）
}

interface RaceResult {
	characterId: string;
	rank: number; // ランダムで決定された順位
}

// bets コレクション
interface Bet {
	id: string;
	raceId: string;
	userId: string; // ベットした人
	targetCharacterId: string; // ベット対象のキャラクターID
	points: number; // ベットポイント
	paidOut?: boolean; // 配当済みフラグ
	payout?: number; // 配当額
	odds?: number; // 確定オッズ
	createdAt: Timestamp;
	updatedAt: Timestamp;
}

// characters コレクション（マスタデータ）
interface Character {
	id: string; // キャラクターID（例："rabbit"）
	name: string; // 表示名（例："ウサギ"）
	emoji: string; // 絵文字（例："🐰"）
	description?: string; // 説明
	order: number; // 表示順
	enabled: boolean; // 有効/無効
}
```

## レースルール

1. 毎日自動でレースが開催される
2. **固定のキャラクター**がレースに参加する（例：5 体のキャラクター）
3. ユーザーは NFC 端末またはアプリでキャラクターにベットできる（1pt〜）
4. 終了時刻（設定可能）にレース終了
5. **勝者はランダムで決定される**（運要素）
6. ベット結果に応じてポイントが変動

### キャラクター例

- キャラクター A（例：ウサギ）
- キャラクター B（例：カメ）
- キャラクター C（例：チーター）
- キャラクター D（例：ライオン）
- キャラクター E（例：オオカミ）

## 配当計算例

- 1 位にベット → 2 倍
- 2 位にベット → 1.5 倍
- 3 位にベット → 1.2 倍
- 4 位以下 → 没収
