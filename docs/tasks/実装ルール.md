# 実装ルール

このドキュメントは、☆ 部室王 ☆ プロジェクトの実装を進める際のルールを定義します。

## ルール

### 1. できないことの扱い

AI ができない作業（ハードウェア連携、外部サービスの設定など）については、**詳しい手順を説明**してください。

**例:**

- Firebase Console での設定
- NFC ハードウェアとの連携
- Vercel へのデプロイ設定
- 環境変数の設定

これらは人間が実行する必要があるため、以下の形式で説明すること:

```markdown
## 【手動作業】Firebase Console での設定

以下の手順で Firebase Console から設定を行なってください:

1. Firebase Console (https://console.firebase.google.com/) にアクセス
2. プロジェクトを選択
3. ...（詳細な手順）
```

### 2. UI での絵文字使用禁止

UI コンポーネント、画面、テキストなどで**絵文字を使用しない**でください。

**NG 例:**

```tsx
<button>🏠 ホーム</button>
<h1>👑 社長席</h1>
<p>💸 ポイントを送る</p>
```

**OK 例:**

```tsx
<button>ホーム</button>
<h1>社長席</h1>
<p>ポイントを送る</p>
```

代替案:

- アイコンライブラリ（lucide-react など）を使用
- テキストのみで表現
- SVG アイコンを使用

## 実装の進め方

1. タスクファイル（`00_初期セットアップ.md` など）を参照
2. チェックリストに従って実装
3. 手動作業が必要な箇所は明確に指示
4. 実装完了後、関連するチェックリストを更新
