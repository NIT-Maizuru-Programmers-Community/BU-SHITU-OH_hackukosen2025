# タスク00: 初期セットアップ

## 概要

プロジェクトの基盤となる環境構築と共通設定を行う

## 前提条件

- Node.js がインストールされていること
- Firebase プロジェクトが作成済みであること

## タスク一覧

### 1. Firebase セットアップ

- [ ] Firebase プロジェクトの作成
- [ ] Firebase SDK のインストール (`firebase`, `firebase-admin`)
- [ ] Firebase 設定ファイルの作成 (`lib/firebase.ts`)
- [ ] 環境変数の設定 (`.env.local`)
  - `NEXT_PUBLIC_FIREBASE_API_KEY`
  - `NEXT_PUBLIC_FIREBASE_AUTH_DOMAIN`
  - `NEXT_PUBLIC_FIREBASE_PROJECT_ID`
  - `NEXT_PUBLIC_FIREBASE_STORAGE_BUCKET`
  - `NEXT_PUBLIC_FIREBASE_MESSAGING_SENDER_ID`
  - `NEXT_PUBLIC_FIREBASE_APP_ID`

### 2. shadcn/ui セットアップ

- [ ] shadcn/ui の初期化 (`npx shadcn@latest init`)
- [ ] 基本コンポーネントのインストール
  - Button
  - Card
  - Input
  - Avatar
  - Badge
  - Tabs

### 3. 共通レイアウト作成

- [ ] モバイルファーストのレイアウトコンポーネント作成
- [ ] ボトムナビゲーション作成
  - ホーム
  - ランキング
  - レース
  - 履歴
  - 設定
- [ ] ヘッダーコンポーネント作成

### 4. Firestore データベース設計

- [ ] コレクション設計
  - `users`: ユーザー情報
  - `points`: ポイント履歴
  - `races`: レース情報
  - `bets`: ベット情報
  - `transfers`: ポイント送金履歴
  - `attendance`: 在室ログ

### 5. 型定義

- [ ] `types/user.ts`: ユーザー型定義
- [ ] `types/point.ts`: ポイント関連型定義
- [ ] `types/race.ts`: レース関連型定義

## 成果物

- Firebase 接続が確立されたプロジェクト
- shadcn/ui が利用可能な状態
- 共通レイアウト・ナビゲーション
- Firestore のコレクション構造
