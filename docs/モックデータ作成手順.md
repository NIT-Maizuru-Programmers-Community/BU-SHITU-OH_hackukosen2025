# モックデータ作成手順

このドキュメントでは、Firebase に必要なモックデータを作成する手順を説明します。

## 作成されるデータ

以下のコレクションにモックデータが作成されます：

1. **attendance** - 在室状況
   - 現在在室中のユーザー（3-5 人）
2. **races** - レース情報
   - 今日のレース（進行中）
   - 過去 5 日分の完了したレース
3. **bets** - ベット情報
   - 各レースに対するベット履歴
   - 完了したレースには結果とペイアウト情報
4. **transfers** - ポイント送金履歴
   - 過去 7 日間の送金履歴（20 件）
   - メッセージ付き/なしの両方を含む

## 前提条件

- `users` コレクションにユーザーデータが存在すること
- Firebase Admin SDK の認証情報が `.env.local` に設定されていること

## 実行手順

### 1. 依存パッケージのインストール

```bash
npm install
```

これにより、`dotenv` と `tsx` がインストールされます。

### 2. 環境変数の確認

`.env.local` ファイルに以下の環境変数が設定されていることを確認してください：

```
FIREBASE_SERVICE_ACCOUNT_KEY={"type":"service_account",...}
```

### 3. スクリプトの実行

```bash
npm run seed
```

### 4. 実行結果の確認

スクリプトが正常に実行されると、以下のような出力が表示されます：

## データの確認

Firebase Console でデータが正しく作成されたか確認してください：

1. [Firebase Console](https://console.firebase.google.com/) にアクセス
2. プロジェクトを選択
3. Firestore Database を開く
4. 以下のコレクションを確認：
   - `attendance`
   - `races`
   - `bets`
   - `transfers`

## トラブルシューティング

### エラー: users コレクションが空です

`users` コレクションにユーザーデータが存在しない場合、デフォルトのユーザー ID が使用されます。先に認証機能を使ってユーザーを作成してください。

### エラー: FIREBASE_SERVICE_ACCOUNT_KEY が設定されていません

`.env.local` ファイルに Firebase Admin SDK の認証情報が正しく設定されているか確認してください。

### エラー: Permission denied

Firebase のセキュリティルールで、Admin SDK からの書き込みが許可されているか確認してください。

## データの削除

モックデータを削除したい場合は、以下のコマンドを実行してください：

```bash
npm run clear-seed
```

このコマンドは以下のコレクションのすべてのドキュメントを削除します：

- `attendance`
- `races`
- `bets`
- `transfers`

実行結果：

## 注意事項

- このスクリプトは開発環境でのテスト用です
- 本番環境では実行しないでください
- 既存のデータに影響を与える可能性があります（同じ ID のドキュメントは上書きされます）
