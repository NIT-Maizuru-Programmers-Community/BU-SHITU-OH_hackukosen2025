# 要件定義書

## プロジェクト名

**☆ 部室王 ☆**

## プロジェクト概要

舞鶴高専プロコン部の部室活性化を目的としたゲーミフィケーション Web アプリケーション

---

## 1. プロジェクトの背景と目的

### 1.1 背景

- 舞鶴高専プロコン部において、部室への部員の来訪が減少している
- 高専周辺は娯楽が少なく、部員のモチベーション向上が課題

### 1.2 目的

- ゲーミフィケーション要素を取り入れることで、部員の部室来訪を促進する
- 部員同士のコミュニケーションと競争心を活性化させる

---

## 2. システム構成

### 2.1 技術スタック

| カテゴリ           | 技術         |
| ------------------ | ------------ |
| フロントエンド     | Next.js      |
| デプロイ           | Vercel       |
| データベース・認証 | Firebase     |
| UI フレームワーク  | Tailwind CSS |
| UI コンポーネント  | shadcn/ui    |

### 2.2 システム形態

- **Web アプリケーション**（モバイルファースト設計）
- **ハードウェア連携**
  - NFC リーダー（ログイン用）
  - マルチインフォメーションディスプレイ（部室設置）

---

## 3. 機能要件

### 3.1 ポイント機能

| 要件 ID | 機能名           | 説明                                     | 優先度 |
| ------- | ---------------- | ---------------------------------------- | ------ |
| F-001   | ポイント付与     | 部室へのログイン時にポイントを付与する   | 高     |
| F-002   | ポイント閲覧     | 個人のポイント数を閲覧できる             | 1      |
| F-003   | ポイント履歴     | 自分のポイント取得・使用履歴を確認できる | 5      |
| F-004   | ポイントリセット | 定期的にポイントをリセットする           | 中     |

### 3.2 レース機能（ベット機能）

| 要件 ID | 機能名         | 説明                                      | 優先度 |
| ------- | -------------- | ----------------------------------------- | ------ |
| F-005   | レース参加     | 毎日ログイン時に賭ける部員を選択する      | 高     |
| F-006   | レース開催     | 毎日終了時刻にレースを自動開催する        | 高     |
| F-007   | レース結果表示 | 本日のレース結果を表示する                | 3      |
| F-008   | ポイント変動   | レース結果に応じてポイントを付与/減少する | 高     |

**レースルール詳細:**

- ベット可能ポイント: ログインボーナスの 1pt〜
- 結果に応じてポイントが変動

### 3.3 ランキング機能

| 要件 ID | 機能名         | 説明                                 | 優先度 |
| ------- | -------------- | ------------------------------------ | ------ |
| F-009   | リーダーボード | 部員のポイントランキングを表示する   | 4      |
| F-010   | 社長席権利     | トップの部員に社長席利用権を付与する | 中     |

### 3.4 スパチャ機能（ポイント送金）

| 要件 ID | 機能名             | 説明                                                         | 優先度 |
| ------- | ------------------ | ------------------------------------------------------------ | ------ |
| F-011   | ポイント送金       | 他の部員にポイントを送ることができる                         | 中     |
| F-012   | メッセージ付き送金 | 送金時にメッセージを添付できる（例：〜について教えて欲しい） | 低     |

### 3.5 認証・ログイン機能

| 要件 ID | 機能名           | 説明                               | 優先度 |
| ------- | ---------------- | ---------------------------------- | ------ |
| F-013   | NFC 登録         | IC カードの登録動作                | 2      |
| F-014   | NFC ログイン     | NFC リーダーでの部室ログイン       | 高     |
| F-015   | ログインボーナス | ログイン時にボーナスポイントを付与 | 高     |

### 3.6 リアルタイム情報機能

| 要件 ID | 機能名       | 説明                                 | 優先度 |
| ------- | ------------ | ------------------------------------ | ------ |
| F-016   | 在室状況表示 | 部室にいる人をリアルタイムで表示する | 6      |

## 4. 非機能要件

### 4.1 ユーザビリティ

- モバイルファーストの UI 設計
- 直感的で使いやすいインターフェース
- レスポンシブデザイン対応

### 4.2 パフォーマンス

- リアルタイム更新（在室状況、ランキング等）
- 低遅延での NFC ログイン処理

### 4.3 可用性

- Vercel による安定したホスティング
- Firebase によるリアルタイムデータ同期

### 4.4 セキュリティ

- Firebase Authentication によるユーザー認証
- 適切なアクセス制御

## 5. 開発優先順位

| 順位 | 機能                               | 概要                       |
| ---- | ---------------------------------- | -------------------------- |
| 1    | 個人のポイント数閲覧               | 基本機能として最優先       |
| 2    | IC の登録動作(NFC)                 | ハードウェア連携の基盤     |
| 3    | 本日のレース結果                   | ゲーミフィケーションの核心 |
| 4    | リーダーボード(ランキング)         | 競争要素の可視化           |
| 5    | 自分のポイント履歴                 | ユーザー体験の向上         |
| 6    | 部室にいる人がリアルタイムでわかる | コミュニティ機能           |

## 8. 用語定義

| 用語             | 定義                                     |
