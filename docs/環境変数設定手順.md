# 環境変数設定手順

API を動作させるために、以下の環境変数を設定する必要があります。

## 1. `.env.local` ファイルを作成

プロジェクトルートに `.env.local` ファイルを作成してください：

```bash
touch .env.local
```

## 2. 環境変数の設定

以下の内容を `.env.local` に追加してください：

```bash
# Firebase Admin SDK (サーバーサイド用)
FIREBASE_ADMIN_PROJECT_ID=your-project-id
FIREBASE_ADMIN_CLIENT_EMAIL=your-service-account-email@your-project-id.iam.gserviceaccount.com
FIREBASE_ADMIN_PRIVATE_KEY="-----BEGIN PRIVATE KEY-----\nYOUR_PRIVATE_KEY_HERE\n-----END PRIVATE KEY-----\n"

# ハードウェア用 API Key
# セキュアなランダム文字列を生成してください
# 例: openssl rand -base64 32
HARDWARE_API_KEY=your-secure-random-api-key-here

# Firebase クライアント設定（フロントエンド用）
NEXT_PUBLIC_FIREBASE_API_KEY=your-api-key
NEXT_PUBLIC_FIREBASE_AUTH_DOMAIN=your-project-id.firebaseapp.com
NEXT_PUBLIC_FIREBASE_PROJECT_ID=your-project-id
NEXT_PUBLIC_FIREBASE_STORAGE_BUCKET=your-project-id.appspot.com
NEXT_PUBLIC_FIREBASE_MESSAGING_SENDER_ID=your-messaging-sender-id
NEXT_PUBLIC_FIREBASE_APP_ID=your-app-id
```

## 3. Firebase Admin SDK の認証情報を取得

### 手順

1. **Firebase Console にアクセス**

   - https://console.firebase.google.com/
   - プロジェクトを選択

2. **サービスアカウントキーを生成**

   - 左側メニューから「プロジェクトの設定」（歯車アイコン）をクリック
   - 「サービス アカウント」タブを選択
   - 「新しい秘密鍵の生成」ボタンをクリック
   - 警告が表示されたら「キーを生成」をクリック
   - JSON ファイルがダウンロードされます

3. **環境変数に設定**

   ダウンロードした JSON ファイルの内容を確認：

   ```json
   {
     "type": "service_account",
     "project_id": "your-project-id",
     "private_key_id": "...",
     "private_key": "-----BEGIN PRIVATE KEY-----\n...\n-----END PRIVATE KEY-----\n",
     "client_email": "firebase-adminsdk-xxxxx@your-project-id.iam.gserviceaccount.com",
     ...
   }
   ```

   以下の値を `.env.local` に設定：

   - `FIREBASE_ADMIN_PROJECT_ID` → `project_id` の値
   - `FIREBASE_ADMIN_CLIENT_EMAIL` → `client_email` の値
   - `FIREBASE_ADMIN_PRIVATE_KEY` → `private_key` の値（**そのままコピー、改行は `\n` のまま**）

   **重要**: `private_key` は必ずダブルクォーテーションで囲んでください。

   例：

   ```bash
   FIREBASE_ADMIN_PRIVATE_KEY="-----BEGIN PRIVATE KEY-----\nMIIEvgIBADANBgkqhkiG9w0BAQEFAASCBKgwggSkAgEAAoIBAQC...\n-----END PRIVATE KEY-----\n"
   ```

## 4. ハードウェア用 API Key を生成

セキュアなランダム文字列を生成します：

```bash
openssl rand -base64 32
```

出力された文字列を `HARDWARE_API_KEY` に設定してください。

例：

```bash
HARDWARE_API_KEY=Xp9Kj2mN5qR8sT1vU4wY7zA0bC3dE6fG
```

**この API Key はハードウェア（NFC リーダー等）からのリクエストに使用されます。**

## 5. Firebase クライアント設定を取得

### 手順

1. **Firebase Console にアクセス**

   - https://console.firebase.google.com/
   - プロジェクトを選択

2. **ウェブアプリの設定を確認**

   - 左側メニューから「プロジェクトの設定」（歯車アイコン）をクリック
   - 「全般」タブを選択
   - 下にスクロールして「マイアプリ」セクションを確認
   - ウェブアプリが未登録の場合は「アプリを追加」→ ウェブ（`</>`）アイコンをクリック

3. **設定値をコピー**

   表示される `firebaseConfig` オブジェクトから以下をコピー：

   ```javascript
   const firebaseConfig = {
   	apiKey: "AIzaSy...",
   	authDomain: "your-project-id.firebaseapp.com",
   	projectId: "your-project-id",
   	storageBucket: "your-project-id.appspot.com",
   	messagingSenderId: "123456789",
   	appId: "1:123456789:web:...",
   };
   ```

4. **環境変数に設定**

   ```bash
   NEXT_PUBLIC_FIREBASE_API_KEY=AIzaSy...
   NEXT_PUBLIC_FIREBASE_AUTH_DOMAIN=your-project-id.firebaseapp.com
   NEXT_PUBLIC_FIREBASE_PROJECT_ID=your-project-id
   NEXT_PUBLIC_FIREBASE_STORAGE_BUCKET=your-project-id.appspot.com
   NEXT_PUBLIC_FIREBASE_MESSAGING_SENDER_ID=123456789
   NEXT_PUBLIC_FIREBASE_APP_ID=1:123456789:web:...
   ```

## 6. 設定完了後の確認

### 開発サーバーを起動

```bash
npm run dev
```

### API の動作確認

```bash
# ヘルスチェック（認証不要のエンドポイントで確認）
curl http://localhost:3000/api/ranking

# 正常に動作していれば、ランキングデータが返ってきます
```

## 7. Vercel へのデプロイ時の設定

Vercel にデプロイする場合は、Vercel のダッシュボードから環境変数を設定してください：

1. **Vercel ダッシュボードにアクセス**

   - https://vercel.com/
   - プロジェクトを選択

2. **環境変数を設定**
   - 「Settings」タブを選択
   - 「Environment Variables」セクションを開く
   - `.env.local` に設定した環境変数をすべて追加

**重要**:

- `FIREBASE_ADMIN_PRIVATE_KEY` は改行（`\n`）を含むため、そのままコピーしてください
- 本番環境用の値を設定してください（開発環境と分ける場合）

## セキュリティに関する注意事項

### ⚠️ 絶対に Git にコミットしないこと

- `.env.local` ファイルは **絶対に Git にコミットしないでください**
- `.gitignore` に `.env.local` が含まれていることを確認してください

### 🔒 API Key の管理

- `HARDWARE_API_KEY` はハードウェア開発者にのみ共有してください
- 定期的に変更することを推奨します
- 流出した場合はすぐに新しいキーを生成して更新してください

### 🔐 Firebase サービスアカウントキー

- サービスアカウントキーは強力な権限を持ちます
- 流出した場合は Firebase Console から該当のサービスアカウントを削除し、新しいキーを生成してください

## トラブルシューティング

### エラー: "Firebase admin initialization error"

**原因**: 環境変数が正しく設定されていない

**解決策**:

1. `.env.local` ファイルがプロジェクトルートに存在するか確認
2. 環境変数名が正確に一致しているか確認
3. `FIREBASE_ADMIN_PRIVATE_KEY` がダブルクォーテーションで囲まれているか確認
4. 開発サーバーを再起動

### エラー: "Invalid API Key"

**原因**: ハードウェア用 API Key が設定されていない、または間違っている

**解決策**:

1. `.env.local` に `HARDWARE_API_KEY` が設定されているか確認
2. リクエストヘッダーに `X-API-Key` が含まれているか確認
3. 開発サーバーを再起動

### 環境変数が反映されない

**解決策**:

1. `.env.local` ファイルの保存を確認
2. **開発サーバーを再起動** （重要！）
   ```bash
   # Ctrl+C で停止してから
   npm run dev
   ```
3. ブラウザのキャッシュをクリア
