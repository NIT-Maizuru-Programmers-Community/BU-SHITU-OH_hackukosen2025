# レースキャラクター設定ガイド

☆ 部室王 ☆ のレースシステムで使用するキャラクターの設定方法です。

---

## 📋 概要

レースシステムでは、**固定のキャラクター**がレースに参加します。ユーザーはこれらのキャラクターの中から勝つと思うものを選んでベットします。

---

## 🎭 デフォルトキャラクター

システムにはデフォルトで 5 体のキャラクターが定義されています：

| ID        | 名前     | 絵文字 | 説明           |
| --------- | -------- | ------ | -------------- |
| `rabbit`  | ウサギ   | 🐰     | 素早さが自慢   |
| `turtle`  | カメ     | 🐢     | 着実に進む     |
| `cheetah` | チーター | 🐆     | 最速のスピード |
| `lion`    | ライオン | 🦁     | 百獣の王       |
| `wolf`    | オオカミ | 🐺     | 群れの力       |

## ✏️ キャラクターのカスタマイズ方法

### 1. 既存キャラクターの変更

`types/character.ts` を編集して、名前や絵文字を変更できます：

```typescript
{
	id: "rabbit",
	name: "ウサギ太郎",  // 名前を変更
	emoji: "🐇",          // 絵文字を変更
	description: "超高速ジャンプ",  // 説明を変更
	order: 1,
	enabled: true,
}
```

### 2. 新しいキャラクターの追加

配列に新しいキャラクターを追加します：

```typescript
export const DEFAULT_CHARACTERS: Character[] = [
	// ... 既存のキャラクター
	{
		id: "dragon", // 一意のID
		name: "ドラゴン",
		emoji: "🐉",
		description: "空を飛ぶ",
		order: 6, // 表示順
		enabled: true,
	},
];
```

### 3. キャラクターの無効化

特定のキャラクターを一時的に使用しない場合：

```typescript
{
	id: "wolf",
	name: "オオカミ",
	emoji: "🐺",
	description: "群れの力",
	order: 5,
	enabled: false,  // false に設定
}
```

## 📝 キャラクター ID の命名規則

### ルール

- **小文字英数字のみ**を使用
- **ハイフン** (`-`) や **アンダースコア** (`_`) は使用可能
- **スペースや特殊文字は不可**

### 良い例

- `rabbit`
- `cheetah`
- `red_fox`
- `blue-bird`

### 悪い例

- `Rabbit` （大文字を含む）
- `レッドフォックス` （日本語）
- `red fox` （スペースを含む）
- `fox!` （特殊文字を含む）

## 🎨 絵文字の選び方

### 推奨する絵文字

- 🐰 🐢 🐆 🦁 🐺 （動物）
- 🚗 🚕 🚙 🏎️ 🚓 （乗り物）
- 🚀 ✈️ 🛸 🚁 🛩️ （空を飛ぶもの）
- ⚡ 🔥 💨 ⭐ 💫 （エフェクト）

### 注意点

- **モバイルで表示されるか確認**してください
- **色付きの絵文字**を推奨（白黒は避ける）
- **複数の絵文字を組み合わせない**（例：🐰🏃‍♂️ は避ける）

## 📊 レースへの反映

### 自動反映

キャラクターを変更すると、次回のレース作成時に自動的に反映されます：

1. `types/character.ts` を編集
2. アプリを再デプロイ（Vercel）
3. 次回のレース作成時に新しいキャラクターが使用される

### 手動でレースを作成する場合

```typescript
import { getEnabledCharacters } from "@/types/character";

// 有効なキャラクターを取得
const characters = getEnabledCharacters();

// レースドキュメントに追加
const raceData = {
	characters: characters.map((char) => ({
		characterId: char.id,
		name: char.name,
		emoji: char.emoji,
		totalBets: 0,
		totalBetPoints: 0,
	})),
	// ... その他のフィールド
};
```

## 🔍 キャラクター情報の取得

### ヘルパー関数

`types/character.ts` には便利な関数が用意されています：

```typescript
import {
	getCharacterById,
	getEnabledCharacters,
	DEFAULT_CHARACTERS,
} from "@/types/character";

// 全キャラクターを取得
const allCharacters = DEFAULT_CHARACTERS;

// 有効なキャラクターのみ取得
const enabled = getEnabledCharacters();

// IDでキャラクターを検索

console.log(rabbit?.name); // "ウサギ"
```

## 🎯 ベストプラクティス

### 1. キャラクター数

- **推奨: 5〜8 体**
- 少なすぎる（3 体以下）: オッズが偏りやすい
- 多すぎる（10 体以上）: ユーザーが選びにくい

### 2. バランス

- 各キャラクターに**個性**を持たせる
- 絵文字は**バラエティ豊か**に
- 説明文で**キャラ付け**をする

## ❓ よくある質問

### Q: キャラクターを追加したら、既存のレースに影響はありますか？

A: いいえ、既存のレースには影響しません。新しいキャラクターは**次回作成されるレース**から使用されます。

### Q: キャラクターの画像を使いたいのですが？

A: 現在は絵文字のみ対応しています。画像を使用したい場合は、以下を実装する必要があります：

1. キャラクター画像をアップロード（例：`/public/characters/`）
2. `Character` インターフェースに `imageUrl` フィールドを追加
3. UI コンポーネントで画像を表示するように変更

### Q: レース中にキャラクターを変更できますか？

A: できません。レースが作成された時点でキャラクターが固定されます。レース完了後に変更してください。

### Q: キャラクターごとに勝率を変えることはできますか？

A: 現在の実装では、全キャラクターが完全ランダムで当選確率は同じです。勝率を変えたい場合は、`hardware/auto_complete_race.py` の抽選ロジックを変更する必要があります。
