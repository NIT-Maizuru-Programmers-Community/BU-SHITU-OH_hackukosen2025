# BU-SHITU-OH_hackukosen2025
Hack U KOSEN 2025 OSAKA「🤩部☆室☆王🤩 」開発用

## Branch の作り方

本プロジェクトでは，GitHub Flow モデルを採用します．  
[Git 構成管理におけるブランチ戦略ガイド(git-flow と Github Flow)](https://zenn.dev/fumi_mizu/articles/dd3fb628a182f5)  
なお，ブランチへのマージについては，GitHub 上で PullRequest を出してチームメンバーに承認を得てからマージすることを基本とします．(Git コマンドで直マージすることは絶対ダメです．)  
ブランチ名は，featuer/機能名と言った方法で命名を行います．(ブランチ名の前に接頭語[prefix]をつけましょう．)

## コミットメッセージの書き方

コミットメッセージの初めには必ず prefix をつける様にしましょう．

(良い例)

```shell
✨ 機能Aを追加
```

(悪い例)

```shell
プログラム動かなくて死ぬ
```

絵文字は以下のサイトから，コピペできます．  
[gitmoji.dev](https://gitmoji.dev/)

#### 機能開発・改善

|   絵文字   | コード       | 説明                     |
| :--------: | :----------- | :----------------------- |
| :sparkles: | `:sparkles:` | 新機能の追加             |
|   :zap:    | `:zap:`      | パフォーマンス改善       |
|   :art:    | `:art:`      | コードの構造・書式の改善 |
| :recycle:  | `:recycle:`  | リファクタリング         |

#### テスト

|       絵文字       | コード               | 説明                                       |
| :----------------: | :------------------- | :----------------------------------------- |
| :white_check_mark: | `:white_check_mark:` | テストの追加・更新、またはテストが通ること |

#### バグ修正

|   絵文字    | コード        | 説明                         |
| :---------: | :------------ | :--------------------------- |
|    :bug:    | `:bug:`       | バグ修正                     |
| :ambulance: | `:ambulance:` | 緊急修正（ホットフィックス） |

#### ドキュメント

| 絵文字  | コード    | 説明                     |
| :-----: | :-------- | :----------------------- |
| :books: | `:books:` | ドキュメントの追加・更新 |

#### インフラ・設定

|        絵文字         | コード                  | 説明                                           |
| :-------------------: | :---------------------- | :--------------------------------------------- |
| :construction_worker: | `:construction_worker:` | CI（継続的インテグレーション）関連の追加・更新 |
|       :bricks:        | `:bricks:`              | インフラ関連の変更                             |
|       :wrench:        | `:wrench:`              | 設定ファイルの追加・更新                       |
|   :heavy_plus_sign:   | `:heavy_plus_sign:`     | 依存パッケージの追加                           |
